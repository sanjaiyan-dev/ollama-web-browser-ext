## Overview

This UI design system establishes a "wow-factor" visual language for a modern browser extension housing a local AI agent (Ollama). The aesthetic bridges **Liquid Glass** (dynamic, adaptive transparency that reflects the environment) with **Aurora UI** (dreamy, flowing ambient gradient meshes). 

Because the interface lives in a restricted Chrome extension popup (typically up to `w-400px` and `h-600px`), it must feel like a premium, native floating window. The interface is highly interactive; tactile micro-animations (bouncy scaling and spring physics) provide immediate satisfaction, making local AI inference feel lightweight, fast, and futuristic.

## Colors

The palette embraces a deep, dark atmospheric base juxtaposed with intensely vibrant, ambient neon accents driven by the Aurora UI trend.

- **Background Base (#05050A):** A near-black void that allows glass components to pop.
- **Aurora Accents (#00E0FF, #8B5CF6, #FF2E63):** Used purely as blurred background meshes or glowing text gradients to simulate energy and AI "thinking" states.
- **Surfaces:** Translucent colors applied with `backdrop-filter` in Tailwind.
- **Text:** High-contrast cool whites for legibility on dark glass.

*Tailwind Implementation Tip:* Map these to your `tailwind.config.js` and use `bg-white/5` or `bg-surface-glass` alongside `backdrop-blur-xl`.

## Typography

- **Headlines (Plus Jakarta Sans):** Used for titles, AI agent names, and buttons. It offers a geometric, friendly, yet highly technical vibe.
- **Body (Inter):** Used for AI conversation outputs and code blocks. Highly readable for long-form reasoning generated by Ollama.
- **Labels:** Uppercase, tracked out (`tracking-widest` in Tailwind) for metadata like token counts, local models (e.g., `GEMMA 4 E4B`), or timestamps.

## Layout

The extension uses a **Floating Spatial Layout**. Elements should not bleed to the harsh edges of the Chrome popup window. 

- **Container Constraints:** Max width of `400px`, max height of `600px`. 
- **The "Floating" Canvas:** Give the main outer wrapper a `p-4` padding so the internal content appears to float seamlessly in the middle.
- **Ambient Background:** Place 2-3 fixed `div`s with `blur-[100px]` and Aurora colors behind the main layout. They should slowly rotate or pulse using CSS keyframes.
- **Bottom Offset:** Ensure all scrolling conversation lists use `pb-[80px]` (or the `nav-offset` token) so the last message isn't hidden behind the floating bottom navigation.

## Elevation & Depth

Liquid Glass completely abandons harsh, opaque drop shadows in favor of **refractive depth**.

- **Level 1 (Background):** The Aurora mesh gradients on the deep background base.
- **Level 2 (Panels):** AI chat bubbles and cards. They use `backdrop-blur-md bg-white/[0.04] border border-white/[0.08]`. 
- **Level 3 (Interactive Layers):** The floating bottom nav bar. Uses aggressive blurring (`backdrop-blur-2xl bg-white/[0.08]`) and a slightly brighter border (`border-white/20`) to create a clear z-index separation.
- **Shadows:** Use large, highly diffused, colored shadows (e.g., `shadow-[0_8px_32px_rgba(139,92,246,0.15)]`) instead of harsh black drop shadows.

## Shapes

The interface completely avoids sharp corners to create a liquid, fluid feel.
- **Containers & Cards:** Use `{rounded.lg}` (`16px` or `24px` Tailwind equivalents `rounded-2xl` / `rounded-3xl`).
- **Interactive Elements:** Buttons, the bottom nav, and input fields must be perfectly pill-shaped `{rounded.pill}` (`rounded-full`).

## Components

- **Bottom Nav (Glass Effect):**
  A floating pill anchored to the bottom of the extension (`absolute bottom-4 left-4 right-4`). It contains routing icons (Chat, Models, Settings) and the primary user input. 
  *Styling:* `flex items-center justify-between rounded-full bg-white/[0.05] border border-white/10 backdrop-blur-2xl p-2`.

- **Aurora Button:**
  Used for primary actions like "Generate" or "Run Tool." 
  *Styling:* Background uses a subtle `linear-gradient`, but the *wow* factor comes from a pseudo-element behind it that pulses with a `blur-lg` effect, acting as a glowing aura. 

- **Agent Message Bubble (Glass Card):**
  *Styling:* Standard Liquid Glass aesthetic. Include a micro-interaction where the border glows subtly based on the mouse position (achievable via a simple React `onMouseMove` tracking script that applies a radial gradient background).

- **Animations & Micro-animations:**
  - **Hover:** All buttons should have `transition-all duration-300 ease-[cubic-bezier(0.23,1,0.32,1)]`.
  - **Click:** Use Framer Motion for spring physics (`whileTap={{ scale: 0.95 }}`). 
  - **AI Thinking:** When Ollama is inferring, animate the Aurora background meshes to spin and expand smoothly, giving a visual representation of "computation."

## Do’s and Don’ts

- **Do** use `contain: strict` on elements with heavy blurs (like the bottom nav) to prevent Chrome extension rendering lag/dropped frames.
- **Don't** use standard black or gray borders. Always use semi-transparent white (e.g., `border-white/10`) to create the physical "edge" of the glass.
- **Do** constrain text contrast. Glassmorphism easily ruins readability. Ensure text on glass elements hits WCAG AA standards by dimming the background mesh directly beneath text blocks.
- **Don't** clutter the minimal Chrome extension estate. If the AI outputs a massive code block, collapse it behind a clean, glass "Show Code" accordion.
- **Do** utilize Framer Motion or CSS Spring effects for opening tools or navigating between views, to make the local connection feel native and liquid.
